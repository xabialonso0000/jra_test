# 目的
競馬において回収率（利益）を最大化する予測システムを構築する

# 指標
回収率 = 払戻金合計 ÷ 購入金額合計

# 重要指標
期待値 = 予測勝率 × オッズ

# 目標
期待値 > 1 の馬のみ購入し、長期的に回収率 > 1 を目指す

# 予測対象
各馬の勝率（1着確率）

# 使用データ
・レース情報（開催日、距離、馬場、競馬場）
・出走馬情報（馬名、騎手、枠順など）
・過去成績（直近レース）
・オッズ（単勝）

# 制約
・未来情報を使用しない（データリーク禁止）
・購入時点のオッズを使用する


# ① データ取得
- Yahooスポーツ（競馬）から以下を取得
  ・レース情報
  ・出走馬情報
  ・オッズ
  ・過去成績（リンク先）

# ② データ保存
- テーブル構成
  ・races
  ・horses
  ・results
  ・odds

# ③ 前処理
- 欠損値補完
- カテゴリ変換（騎手・競馬場）
- 数値正規化
- 時系列整形（未来情報を含めない）

# ④ 特徴量作成
- 直近3走の着順平均
- 騎手勝率
- 距離適性
- 馬場適性
- 人気平均との差

# ⑤ モデル学習
- LightGBMで学習
- 目的変数：1着=1、それ以外=0
- 出力：勝率（確率）

# ⑥ 期待値計算
- 期待値 = 予測勝率 × オッズ

# ⑦ 購入ロジック
- 期待値 > 1 の馬のみ購入
- 条件追加（任意）
  ・オッズ > 2.0
  ・上位3頭まで

# ⑧ バックテスト
- 過去データで検証
- 指標
  ・回収率
  ・的中率
  ・最大ドローダウン

# ⑨ 本番運用
- レース前にデータ取得
- モデルで予測
- 期待値計算
- 購入判断

# ⑩ 改善
- 外れレース分析
- 特徴量追加
- モデル更新
